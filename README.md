# MedServicePrice.kz — Агрегатор цен на медицинские услуги в Казахстане

🚀 **Рабочий MVP** системы умного поиска, мониторинга и аналитики цен на медицинские услуги, анализы и приёмы врачей в крупнейших городах Казахстана (Астана, Алматы и др.).

Проект разработан в рамках хакатона и решает проблему прозрачности цен на медицинском рынке, агрегируя данные из различных источников (клиники, лаборатории, медицинские справочники) и приводя их к единому стандарту.

---

## 🔥 Ключевые фичи MVP
* **Умный поиск с аналитикой:** Поиск услуг по справочнику (с учетом синонимов) с мгновенным расчетом минимальной, максимальной и средней цены по городу.
* **Автодополнение (Suggest):** Быстрые подсказки при вводе в поисковую строку.
* **Тайм-лимит на актуальность:** Система автоматически отсекает данные старше 30 дней (согласно ТЗ).
* **Умная батч-загрузка филиалов:** Оптимизированная выдача адресов клиник (схлопывание гигантских списков филиалов крупных агрегаторов для стабильности UI).
* **История цен и тренды:** Эндпоинт для построения графиков изменения стоимости услуг за последние 30 дней с расчетом направления тренда (Up/Down/Stable).
* **Автоматический планировщик (Scheduler):** Запуск парсеров по расписанию (`APScheduler`) в ночное время с последующей нормализацией данных.
* **Админ-панель разметки:** Очередь неразмеченных сырых услуг для ручной привязки оператором к эталонному каталогу.

* **Backend:** Python 3.10+, FastAPI, SQLAlchemy (ORM)
* **Frontend** * Vue.js 3 Tailwind CSS Leaflet.js 1.9.4 Font Awesome 6  Web Storage API HTML5 / CSS3
* **Database:** PostgreSQL / SQLite (в зависимости от конфигурации окружения)

## 🚀 Инструкция по запуску

### 1. Клонирование репозитория

```bash
git clone [https://github.com/xtron11/MedServicePrice]
cd medserviceprice

```

### 2. Настройка окружения

Создайте файл `.env` в корневом каталоге проекта и заполните данными для подключения

```env
DB_HOST=
DB_PORT=
DB_USER=
DB_PASS=
DB_NAME=

```

### 3. Запуск через Docker Compose (Рекомендуемый способ)

Развертывание всей инфраструктуры (API, База данных, Планировщик) одной командой:

```bash
docker-compose up --build -d

```

*API бэкенда станет доступно по адресу:* `http://localhost:8000`

*Интерактивная документация Swagger:* `http://localhost:8000/docs`

## 🕒 Управление парсерами и расписанием

По умолчанию внутри контейнера запускается планировщик `scheduler.py`, который работает по часовому поясу **Asia/Almaty** и запускает сбор данных ночью:

* `02:00` — КДЛ Олимп
* `03:00` — Инвитро
* `03:45` — Helix
* `04:30` — Doq.kz
* `05:00` — Нормализация данных

### Ручной запуск для проверки через Docker
Если вам необходимо протестировать сбор данных или запустить первичный сидинг базы прямо сейчас, не дожидаясь ночи, используйте проверенные команды через `docker compose run`:

```bash
# Запустить первичный сидинг и нормализацию базы данных
docker compose run --rm api python -m back.app.normalizer --seed

# Запустить абсолютно все парсеры и нормализацию последовательно
docker compose run --rm api python back/scheduler.py --now

# Запустить конкретный парсер отдельно
docker compose run --rm api python back/scheduler.py --kdl       # КДЛ Олимп
docker compose run --rm api python back/scheduler.py --invitro   # Инвитро
docker compose run --rm api python back/scheduler.py --doq       # Doq.kz (Врачи)
docker compose run --rm api python back/scheduler.py --helix     # Helix


## 🔗 Основные API Эндпоинты (Эндпоинты MVP)

| Метод | Эндпоинт | Описание |
| --- | --- | --- |
| `GET` | `/api/stats` | Общая статистика системы (кол-во клиник, активных услуг, городов). |
| `GET` | `/api/suggest` | Живой поиск/автодополнение по справочнику при вводе (`?q=...`). |
| `GET` | `/api/search` | Умный поиск с фильтрами по городу, цене, категориям и выдачей аналитики. |
| `GET` | `/api/cities` | Список всех городов, представленных в системе. |
| `GET` | `/api/clinic/{id}/services` | Карточка клиники: контакты, ограниченный список филиалов и весь её прайс. |
| `GET` | `/api/service/history` | История изменения цены конкретной услуги за 30 дней для графиков. |
| `GET` | `/api/admin/unmatched` | Очередь неразмеченных услуг для панели модератора. |
| `POST` | `/api/admin/match` | Ручная привязка сырой услуги к эталонному элементу каталога. |
